Extensible

  • Rules are declared; Carnap figures out how to check them.

    • The proof system can be adapted to work with a variety of textbooks.

    • The proof system can be extended or restricted to encompass a variety of logics.

    • Derived rules can be added by instructors. They could even be added by students (work in progress) while the program is running, to eliminate tedious repetition

  • Languages are introduced as special cases of a general abstract syntax.

    • Many syntactic concepts can be defined at this level once and for all. They're then available in any new language presented as an instance of the abstract syntax.

    • Formal languages used in philosophy, mathematics, computer science, and linguistics can be introduced as instances of this syntax.

Modular

  • The same core can drive a variety of interfaces.

    • A simple interface based on Blockly can be used while students are still learning syntax (work in progress). A more sophisticated interface (like the one in these slides) could be adopted later in the course.

    • A variety of different display-formats (as nested lists, as trees, as circuits) for proofs in a single system could be presented, helping students decouple their concept of proof from a single format.

  • The same interface can present a variety of proof-systems.

    • The nested list interface can be used with any logic presented by a (standard single-conclusion) sequent calculus.

    • This makes it possible to demonstrate hands-on how different sub-classical implications play out; e.g. the equivalence of P_1 ∨ ¬P_1 and (((P_1 → P_2) → P_1) → P_1).

Accessible

  • The program is implemented in a high-level language, then translated to Javascript.

    • A powerful implementation language makes extension and maintenance easier.

    • Javascript guarantees cross-platform and installation-free accessibility.

    • The web files can be downloaded for offline use on any computer that has a modern web-browser.

    • The core could be compiled into a standard application as well for standalone use without a browser.

  • The source code will be available under an open licence on Github.com, where other researchers, teachers, and students can inspect, correct, and extend it.
